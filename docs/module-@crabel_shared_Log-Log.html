    static defHeader :Array.<String>
    Log.js, line 336

    Default log header. Log headers support the following substitution token: 1) {{log.mode}} : Mode in which the log is being initialized (append, replace, unique, unique_pre) 2) {{log.file}} : Name of the log file being used for logging. 3) {{log.level}} : Maximum event level/type being logged (err, warn, info,trace). 4) {{log.echo}} : on/off value indicating whether echo us enbled or not. 5) {{date.now[,format]}} : String representing the current date and time. Optionally a formatting string can be provided.

    static defTimeFormat :string
    Log.js, line 416

    Default format of the timestamp added to log messages, this value is assigned to new instances of the Log object. For information on the format used see the Date.format documentation. This static `property can be used to change the time format of every new Log instance that is creates. It is generally recommended to change this property and not the instance property in order to * provide a consistent look between log files of the same application.

    static err(msg, cbopt, …substopt)
    Log.js, line 574

    Outputs an error message to the log file and/or standard output stream. The timestamp and event type label are pre-pended to the given message. The timestamp can be customized via the Log.timeFormat property; the event type label can be customized via the 'Log.TypeLabel' property. The message is logged using the global log instance.
    Parameters:
    Name Type Attributes Description
    msg String The message to send to the log file and/or console.
    cb LogCallback <optional>
    		 Optional callback function. If provided, it will make the call after the log entry is logged.
    subst * <optional>
    <repeatable>
    		 Optional list of substitution arguments. When present, the method formats the message using either str_util.format or str_util.expand depending on whether a substCallback was provided.

    static makeLogFileName(dir, name) → {string}
    Log.js, line 519

    Builds a log file name with the given name and path. The methods adds the ".log" extension if necessary. If using a unique mode, this function calls the file.makeUnique method to ensure the file name is unique, this method pre-pends a two digit number starting from 00 and incrementing until a file with the same name is not found makeLogFileName ensures that the path leading to the log file exists by calling path.ensure.
